Using Unix Effectively

Introduction

Unix is a very powerful tool. It was made by programmers for themselves to use, and as a result tends to be fast, configurable, extremely powerful. It also can be decidedly daunting for beginners, as it was not designed with beginners in mind. By talking about some of the basics of the Unix philosophy and building blocks of the Unix system and programs, we hope to give you motivation to explore the full power and richness of the system.

The Unix Philosophy
  • Take the time to find the most efficient way to do something.
  • Automate tedium.
  • Don't reinvent the wheel unless you want to.
  • Use all the resources available to find answers to your questions.
For a longer description of the Unix philosophy, look at Eric Raymond's The Art of Unix Programming

Shells

 A shell is a program which takes user input and passes it on to the operating system. Unix gives you a choice in shells (or write your own!). A few different shells:
  • csh, C shell
  • ksh, Korn shell
  • bash, Bourne again shell (default on our system)

What does the shell do for you?

At the basic level a shell allows you to navigate the file system and run programs. It can also be programmed, allowing you to automate and script things.

A simple shell script

This script will take a file, make a backup copy, and then open the file in Emacs. 
#!/bin/sh
# edit -- example shell script

cp $1 $1.bak             # make a copy. $1 is the first command line variable
emacs $1                 # start emacs
Change the permission, allowing the file to the executed, and put it in your path directory, and you've got a working program.

But what's a path?

The path is a list of places that the shell looks when you ask it to run a program. 
[jeff@localhost jeff]$ echo $PATH
/usr/local/bin:/bin:/usr/bin:/usr/X11R6/bin:/usr/games:/opt/home/jeff/bin:/usr/java/jdk1.3/bin/:/opt/home/jeff/lib:/usr/X11R6/bin:/usr/games

Permissions
  • Three kinds of permissions: read, write and execute
  • Three levels of permissions: user, group and global
To see permissions for a file, use ls -l: 
      [jeff@localhost jeff]$ ls -l edit 
      -rw-r--r--    1 jeff     students           33 Mar  1 09:27 edit
These permissions do not apply to root, who has complete and unrestricted access to every file in the system.

Getting Around a Unix Filesystem
  • It's a tree.
  • Everything is a file.
  • Things are stored according to function. A few examples:
    • /bin Binary (executable files)
    • /doc Documentation
    • /usr User added files
    • /src Source code
    • /etc Configuration files
  • Symbolic links are like shortcuts, they create a weak link from one point in the file tree to another. Let's say there is a directory ~/code/java/ps2/prob2/src/, and you don't want to type it in every time you go there. A symbolic link can be created.
    bash$ ln -s ~/code/java/ps2/prob2/src current
    Now current is a symbolic link to the same place.

Pipes and Redirects

 Unix makes it easy to use the output of one program as the input of another. Combine this with the fact that everything in the system is a file, and you have some powerful options. A few examples of pipes:
  • ps aux | grep jeff
  • cat file.java | lpr
  • who | cut -c1-8 | sort -u | pr -l1 -8 -w78 -t
